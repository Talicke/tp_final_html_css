Bienvenue sur la section HTML:

Le HTML c'est quoi ce truc?

HTML permet de Structure les pages à l’aide de balise, c’est le squelette d’une page.
C'est un langage interprété par le navigateur

Quelles sont les bonnes pratiques:

- L’indentation :
o Lorsqu’une balise contient une autre balise, on effectue une tabulation. C’est ce qu’on appelle l’indentation. Ce n’est pas obligatoire au fonctionnement, mais c’est indispensable à la compréhension humaine.
- Le camelCase :
o En informatique, on utilise la notation camelCase consiste à écrire en minuscule, sans accents, sans espace et sans ponctuation en séparant les mots par des majuscules.
o Ex: Mon titre => monTitre // Une périphrase => unePeriphrase
- Les commentaires :
o Le texte écrit entre des balises de commentaire sera ignoré par le navigateur et invisible sur votre page. C’est un mémo pour développeur.
o Exemple < !—Personne ne pourra lire ceci sur ma page-->
- Notre code :
o Information importante, tout ce que vous écrivez en HTML sera lisible par tout le monde en affichant le code source de notre page. Les commentaires y compris, attention à ne pas écrire n’importe quoi !
- Espace de travail :
o Ne mettez pas tous vos projets sur votre bureau dans des « nouveaux dossiers ». Créez un espace de travail propre, ordonné et nommé vos fichiers et vos dossiers de manière logique.

La structure minimum d'une page HTML:

Ces balises sont obligatoires dans toutes les pages HTML :
o html : contient toutes les balises HTML.
o head : En-tête de notre page, elle contient à minima la balise title qui affiche le titre, mais souvent beaucoup d’autres balises.
o body : le corps de notre page, elle contient l’intégralité de notre contenu de page (articles ,images ,liens ,vidéos ,…)

Il existe deux types de balise

Les balises par paire:

Elles contiennent du texte, encadré par deux balises. Elles indiquent au navigateur le début et la fin de notre texte, ainsi que sa nature. Ici, la balise h1 est une balise de titre, le navigateur sait donc que notre texte, encadré par nos balises, est un titre. La balise fermante contient toujours un slash «/ » après le chevron.

Les balises orphelines:

Ces balises permettent le plus souvent d’insérer un élément dans notre page. Ici, la baise br permet de faire un retour à la ligne. Par convention, il est préférable de mettre un slash « / » à la fin de notre balise, comme ceci «/br», même si ce n’est pas obligatoire.
